# Hypo-Research First-Principles Review Design Spec

## 背景

Hypo-Research 当前是一个学术科研辅助工具包，包含文献检索、引用图扩展、LaTeX 写作检查、引用验证、模拟审稿、rebuttal、科研创意、项目管理和会议纪要等 19 个 Skill/CLI 能力。用户希望从第一性原理重新审查其产品定位、与 Hypo-Workflow 的关系、指令入口、接口设计、PDF/全文阅读、文献调研报告和长期路线。

## 目标

1. 产出一份战略/架构审查报告。
2. 将审查结论转成后续可执行路线图。
3. 比较 Hypo-Research 与 Hypo-Workflow 的四种关系预设：融合进去、作为外挂插件、底层依赖、完全独立。
4. 重新设计更自然的少数固定场景入口，降低用户记忆 `$hypo-xxx` 的负担。
5. 调研 PDF 论文转 MD/HTML、图片/表格识别和分级全文阅读方案。
6. 优化文献调研报告和陌生领域引导，并形成 research OS / PhD workflow 长期蓝图。

## 非目标

- 本 Cycle 不做主项目大规模代码实现。
- 不把 PDF 解析做成完整产品级能力。
- 不做大而全竞品市场报告，只提炼对产品和架构有用的启发。
- 不把审稿级深度自动化作为短期承诺。

## 关键约束

- 主要服务对象是 PhD/研究生，兼顾独立研究者。
- 与 Hypo-Workflow 关系评估中特别重视对非研究用户的污染程度。
- PDF/全文深读优先回答：用了什么方法、跑了什么数据、关键图表说明什么、claim 是否被实验支持。
- M03 原型验证开始前，必须先提醒用户提供 2 篇 PDF。
- 可以产出可合并的架构/接口改造方案，但代码实现另开 Cycle。

## 验证方式

- 代码阅读和模块边界分析。
- 产品推理和用户场景走查。
- 轻量竞品/工具调研：检索类工具、PDF 转 MD/HTML 与图表/表格抽取方案。
- 1-2 篇用户提供 PDF 的轻量流程验证，如条件允许。

## 输出

- 四份 Milestone 分析报告。
- 最终战略/架构审查报告。
- 新入口信息架构草案。
- PDF/全文阅读接口与 pipeline 草案。
- 近期、中期、长期路线图。
